Hybrid Apps

Most people want to use ChocolateChip-UI to make hybrid apps. Since ChocolateChip-UI provides themes for iOS, Android and Windows Phone, those are the three targets for hybrid apps.

iOS

Because of restrictions by Apple, you need to be on a Mac to make and test a hybrid app for iOS. You can easily use Xcode to create a Web View for your hybrid app. We have this process documented in the following blog post. But rather than following that route, we've create an NPM module to make this easier. Just run a command line tool that creates the project for you. Here's how.

NPM

On your Mac, open the terminal and run the following command:

sudo npm install -g chui-ios

This will install our NPM module globally so that it's always available to run at all times. When you run this, you will need to enter the user password for your computer. After this is instalaled, you can run it in your terminal. Try running the following:

chui-ios

You should see the following returned:

$: chui-ios
Usage: --proj "myApp" --username "Joe" --chuiapp "~/Documents/myWebApp"

Options:
  --proj                                                              [required]
  --username                                                          [required]

Missing required arguments: proj, username

chui-ios accepts four arguments (the double hyphens are required):

  • --proj (the name of your project)
  • --username (your username on your computer)
  • --chuiapp (path to an existing ChUI Web app, otherwise it will create a default html page)
  • --appicons (path of icons for the hybrid path)

To successfully create a project with this command line tool, you'll need to give your project a name, a single word name is best, and your username, that is, the name you use on your computer. Xcode needs to know what your username is so that it can find the frameworks it needs in your computer's file system. Not providing the username you use on your computer can result in a non-functional Xcode project.

Adding the paths to an existing Web app and its icons is easy, type the argument name first, followed by a space. Then drage the folder on to the terminal. The path to that folder will appear after the argument. Here an example of how this command might look:

chui-ios --proj "Hamburger" --username "Wimpy" --chuiapp /Users/bombo/Documents/hamburger --appicons /Users/bombo/Desktop/hambuger-icons

Hitting "Return" will execute this command, resulting in an excode project created on the desktop with the name Hamburger. Inside you would find normal Xcode files, a folder called Website with your Web app files inside, and your icons, iTunes are and launch images already in place in Xcode. Double click the Xcode project icon will launch Xcode. Once that is done you can immediate run the project on the simulator, or directly on a tethered iPhone.

You can find out more about this NPM module for iOS hybrid apps on NPM.

Windows Phone

Making a hybrid app for Windows is so simple that we cannot even consider wasting time creating a command line tool for that. Microsoft has made this really easy using Visual Studio. If you already have Visual Studio, you are ready to go, otherwise, we recommend you download Visual Studio Community Edition. It's completely free.

Once you have Visual Studio installed, open it. Click "New Project...". Open the Visual C# tree of templates in the right side column of available templates. Then select "Store Apps" and "Windows Phone App and choose "WebView App (Windows Phone) in the center screen. Then click OK. This will create your WebView project. The resulting project will have a folder call Assets. These are your icons, etc. Replace these with replacements using your own artwork. There is also a folder called HTML. This is where your app goes. You can literally just copy and replace whatever is in there with your Web app. Just make sure your app's main file is called "index.html". After that, you're ready to run. You can read more a more detailed explanation on our blog.

Android

You can install the NPM package to enable creating hybrid apps for Android. This command-line tool works on Mac OS X and Windows.

Mac Install

Open the terminal and enter:

sudo npm install -g chui-android

After hitting return, you will be prompted to enter your password. When it is done you are ready to start creating Android hybrid apps. This does have two dependencies: Orancle's Java and Google's Android Studio.

The chui-android command-line tool can accept four arguments. The first to are required:

  • --proj (the name for your app)
  • --user (your username)
  • --app (path to your ChUI Web app - this is optional)
  • --icons (path to a folder of your app's Android icons - this is optional)

If you run chui-android in the terminal, you will see:

$: chui-ios
Usage: --proj "myApp" --username "Joe" --chuiapp "~/Documents/myWebApp"

Options:
  --proj                                                              [required]
  --username                                                          [required]

Missing required arguments: proj, username

If you leave out --app or --icons, the package will create a default Web page that the project will load and default icons. You can simply replace these at a later time with your final assets. When the command is finished, you will find a folder with your project's name on your desktop.

Open Android Studio and import your project into it. After that you'll be able to run it in the emulolator or deploy it directly to a tethered device.

Open the command prompt and enter:
