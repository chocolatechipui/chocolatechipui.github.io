Paging

ChocolateChip-UI provides a simple way to implement paging trough multiple sections of an article. In general, an article has but a single section, which implement's the vertical scrolling container for the article's content. But if you want to allow the user to access multiple sections of content within the same article, you can use the paging control.

The setup is very simple. You take a normal article and instead of one section, give it as many as you need. The paging control will take care of the paging for you. When using the paging control, each section can have whatever layout is suitable for the content it holds.

<article id="main" class="current paging">
   <section>
      <h2>First Article</h2>
      <ul class='list'>
      <li>
         <h3>Thing One</h3>
      </li>
      </ul>
   </section>
   <section>
      <h2>Second Article</h2>
      <ul class='list'>
      <li>
         <h3>Item One</h3>
      </li>
      </ul>
   </section>
   <section>
      <h2>Third Article</h2>
      <ul class='list'>
         <li>
            <h3>Item 1</h3>
            <cellsubtitle>Item 1 Subtitle</cellsubtitle>
         </li>
      </ul>
   </section>
</article>

In that article's nav put a segmented control with the extra class of paging and a class indicating the orientation for the paging: horizontal or vertical. Since you're putting the segmented control as the last item in the nav, you want to put the class align-flush on it as well:

<div class='segmented paging horizontal align-flush'>
   <a class='button' href='javascript:void(null)' title='previous panel'></a>
   <a class='button' href='javascript:void(null)' title='next panel'></a>
</div>

Next mark the article as capable of paging by putting the paging class on it:

<article id="deserts" class="current paging">

Then you just need to initialize the structure at load time:

$(function() {
   $.UIPaging();
});

And that's all there is to implementing a paging control.
