Editable List

Sometimes you need to give users a way to edit a list, such as changing the order of items or deleting items. ChocolateChip-UI provides the editable list method, $(selector).UIEditList(options), for these purposes. It turns an ordinary list into an editable one. It can accept the following arguments:

  • editLabel: the text shown by the Edit Button, default is "Edit"
  • doneLabel: the text shown by the Done button, default is "Done"
  • deleteLabel: the text shown by the Delete button, default is "Delete"
  • movable: true or false, default is true
  • deletable: true or false, default is true
  • callback, gets executed when the user taps the Done button

If you need to localize the names of the button, you would provide those values as show above. Similarly, if you only want to enable reordering list items, you would pass the movable argument as false. Or if you only wanted to enable deletion, then you would pass the movable argument as false. Otherwise both will be enabled.

When the user enables editing of the list by tapping the "Edit" button, depending on where moving or editing has been enabled, he can modify the list by the possible indicators revealed. When the user taps the "Done" button, the indicators are hidden and any callback gets executed. This callback gets a reference to the editable list. From this reference you can query the list's two properties: data-list-items-deleted and data-list-items-position. If nothing was deleted, the first will be empty. If nothing was moved, the second will return the current, unchanged order of list items.

// Initialize a list for editing:
$('#myList').UIEditList({
  moveable: true,
  deletable: true,
  callback: function(list) {
    console.log('Items deleted: ' + $(list).attr('data-list-items-deleted'));
    console.log('New order of items: ' + $(list).attr('data-list-items-position'));
  }
});

In the above code, notice the list parameter in the callback. This parameter gives us access to the current editable list. Using this we can find out the current value of the two attributes: data-list-items-deleted and data-list-items-position. With this information, our callback can do whatever is need to persist this information either locally or on a remote server.
